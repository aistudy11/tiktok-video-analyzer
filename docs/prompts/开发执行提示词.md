# 开发执行提示词

> 在新的 Claude Code 窗口中使用此提示词，无需上下文即可继续开发。

---

## 使用方法

复制下方提示词，粘贴到新的 Claude Code 会话中执行。

---

## 提示词

```
我需要你帮我继续开发 TikTok 视频分析器项目。

## 项目背景

这是一个跨境电商视频分析工具，核心功能：
1. 分析 TikTok 爆款视频
2. 生成可执行的视频制作脚本（分镜头 + 音乐卡点）
3. 同步到飞书/Notion

## 开发规则（必须遵循）

**在做任何操作前，先阅读以下规则文件：**

1. `.claude/rules/shipany-base.md` - 基座保护规则（最高优先级）
2. `.claude/rules/00-philosophy.md` - 核心开发哲学
3. `.claude/rules/deployment.md` - 部署规范

**核心约束：**
- 基座代码（core/, shared/components/ui/, extensions/）禁止修改
- 业务代码只能在指定目录开发：
  - 后端：`backend/skills/`
  - 前端组件：`themes/default/blocks/tiktok/`
  - Hooks：`shared/hooks/use-tiktok-*.ts` 或 `use-script-*.ts`
  - 页面：`app/[locale]/(landing)/tiktok-*/`

## 执行入口

**阅读 `docs/开发路线图.md`**，这是唯一的开发执行入口。

路线图会告诉你：
1. 当前应该做什么任务
2. 每个任务的文件位置
3. 需要参考的详细设计文档

## 当前阶段

阶段一：视频脚本生成功能

请先阅读开发路线图，然后告诉我当前应该执行哪个任务，等我确认后开始执行。
```

---

## 变体：指定具体任务

如果你已经知道要做哪个任务，可以用这个版本：

```
我需要你帮我开发 TikTok 视频分析器项目的【任务 1.1：创建 script_generator 模块】。

## 开发规则

在做任何操作前，先阅读：
1. `.claude/rules/shipany-base.md` - 基座保护规则
2. `docs/开发路线图.md` - 执行入口
3. `docs/视频脚本生成功能设计.md` - 功能详细设计

## 任务要求

在 `backend/skills/` 下创建 `script_generator/` 模块，包含：
- `__init__.py`
- `generator.py` - 脚本生成核心逻辑
- `prompts/` - Prompt 模板目录
- `schemas.py` - Pydantic 模型

Prompt 模板从 `docs/视频脚本生成功能设计.md` 的第 3 章复制。

请开始执行。
```

